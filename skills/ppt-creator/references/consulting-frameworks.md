# Consulting Frameworks — 構成パターン集

ppt-creator でスライドを組み立てるときの**アウトライン骨格の辞書**。
固定テンプレートではなく、題材に合わせて枚数・順序を調整するための叩き台。

迷ったら **SCQA**。論理的なストーリーが自然に組める汎用フレームワーク。

---

## SCQA — 提案資料・問題解決型

**Situation → Complication → Question → Answer**

聴衆が既に知っている現状 (S) から始め、見落としている課題 (C) を提示し、
意思決定すべき問い (Q) を立て、それに答える形で提案 (A) を示す。

### いつ使うか
- 顧客向け提案資料 (営業・企画)
- 社内での新規施策提案
- 問題解決型のコンサルティング資料

### 標準スライド順 (10〜14 枚)

```
 1. [title]                表紙 (顧客名・日付・発表者)
 2. [通常]                 アジェンダ
 3. [section]              Situation (背景)
 4. [content-image-right]  市場背景 + Mermaid graph TD
 5. [column-layout]        現状 KPI (3 列、各カラムに大きな数字)
 6. [section]              Complication (課題)
 7. [content-image-left]   課題の構造 + 図
 8. [通常]                 顧客の声 (blockquote) + 補足テーブル
 9. [section]              Question → Answer
10. [align-center]         結論 (大きな **強調**)
11. [通常]                 実行計画 (Mermaid Gantt)
12. [通常 / small-text]    Next Steps + 連絡先
```

### コツ
- S は 1〜2 枚でコンパクトに。聴衆は既に知っている
- C で危機感を作る。数値で見せると刺さる
- Q は明示的に書く (「どのツールを採用すべきか？」など)
- A は結論から述べ、根拠を後に

---

## Pyramid Principle — 役員報告・意思決定支援

**結論 → 根拠 → 詳細**

Barbara Minto の Pyramid Principle。最初に結論を 1 枚で示し、
それを 3 本 (原則) の根拠で支え、各根拠を 1〜2 枚で詳細化する。

### いつ使うか
- 役員報告 / 経営決裁
- 時間の無い意思決定者向け
- 「結論から言え」と言われる場面

### 標準スライド順 (5〜7 枚)

```
1. [title]                表紙
2. [align-center]         結論 1 枚 (最大フォント、**強調**)
3. [column-layout]        根拠 3 本柱 (3 列、各柱に数値)
4. [content-image-right]  根拠 ① の詳細 + 図
5. [content-image-left]   根拠 ② の詳細 + 図
6. [通常]                 根拠 ③ の詳細 + テーブル
7. [通常]                 決裁依頼事項 (番号付きリスト + 期日テーブル)
```

### コツ
- Slide 2 は結論だけ。理由は書かない
- 根拠の数は必ず 3 本 (MECE に分ける)
- 各根拠は 1 スライドで独立して読める粒度に

## MECE — 比較・評価

**Mutually Exclusive, Collectively Exhaustive**

評価軸を先に定義し、候補を漏れなく・ダブりなく評価する。

### いつ使うか
- ツール / サービス / 手法の比較
- 選定プロセスの透明性を示したい場面
- ステークホルダーが多く、異論が出やすい意思決定

### 標準スライド順 (8〜12 枚)

```
 1. [title]                表紙
 2. [通常]                 比較の目的と評価軸
 3. [通常]                 評価軸の定義 (テーブル)
 4. [section]              候補一覧
 5. [column-layout]        3 候補の概要 (3 列カード)
 6. [通常 / small-text]    評価マトリクス (大きなテーブル)
 7. [content-image-right]  候補 A の詳細 + アーキ図 (Mermaid)
 8. [content-image-right]  候補 B の詳細 + アーキ図
 9. [content-image-right]  候補 C の詳細 + アーキ図
10. [align-center]         推奨 (**強調** で結論)
```

### コツ
- 評価軸は先に合意を取る (Slide 2-3)
- 評価マトリクスは `small-text` で全軸 × 全候補を 1 枚に収める
- 候補ごとの詳細スライドは**同じレイアウト**で統一すると比較しやすい

---

## 組み合わせの指針

- 提案資料の中で「なぜこの案か」を示すパート → Pyramid
- 比較パートを含む提案資料 → SCQA の A の部分に MECE を埋め込む
- 役員向けの進捗報告 → Pyramid + 簡易 SCQA

構造が決まれば中身はぶれない。**まず構造、次にコンテンツ、最後にレイアウト**。
