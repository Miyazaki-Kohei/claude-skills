# レクターパターンリファレンス

39種のレイアウトパターンの役割・活用シーン・デザイン解説。
パターンの実装コードは `slides/example.md` を参照。

---

## パターン選択ガイド

何を伝えたいかに応じて最適なパターンを選ぶ。

```
何を伝えたいか → 推奨パターン
├── 章の始まり・区切り → Pattern 2 (section)
├── 章のまとめ → Pattern 3 (section-end)
├── 2つの比較・対比 → Pattern 6 (2col比較) or Pattern 38 (対比+結論)
├── 3つの並列要素 → Pattern 8 (3col) or Pattern 9 (3col accent)
├── 4つのフェーズ → Pattern 10 (4col)
├── 段階・成熟度 → Pattern 11 (5col maturity)
├── ステップ・プロセス → Pattern 14 (v-step) or Pattern 15 (横型step)
├── 時系列 → Pattern 16 (timeline)
├── 機能・特徴一覧 → Pattern 17 (icon list) or Pattern 13 (2x3 grid)
├── 画像+テキスト → Pattern 7a/7b (2col text+img)
├── 製品・サービス紹介 → Pattern 22 (card) or Pattern 18 (panel+image)
├── KPI・数値強調 → Pattern 27 (統計) or Pattern 34 (統計比率)
├── 重要メッセージ → Pattern 28 (center-message)
├── 引用 → Pattern 25 (quote)
├── 問いかけ → Pattern 31 (question)
├── まとめ・結論 → Pattern 36 (ガラス風縦並び) or Pattern 38 (対比+結論)
├── データ表 → Pattern 39 (table+highlight)
└── QRコード・リンク共有 → Pattern 30 (QR)
```

## A. タイトル・セクション系

### Pattern 1: タイトルスライド

**役割:** デッキの表紙。プレゼンテーションの第一印象を決める

**活用シーン:**
- プレゼンの冒頭（必須）
- タイトル、サブタイトル、日付、発表者名を提示

**デザインポイント:**
- `<!-- _class: title -->` + `<!-- _paginate: false -->`
- Navy→Teal グラデーション背景、白文字
- h1 でメインタイトル、h2 でサブタイトル
- 日付・発表者名は通常テキスト

**注意事項:**
- タイトルは簡潔に（1-2行）
- 組織名や対象読者はフッターに入れる

**前後の組み合わせ:**
- 後: Pattern 4 (toc) で全体像を示す、または Pattern 2 (section) で最初の章へ

---

### Pattern 2: セクション開始

**役割:** 章の区切り。聴衆に話題の転換を伝える

**活用シーン:**
- 大きなテーマが変わるとき
- 3-5分のまとまりの冒頭

**デザインポイント:**
- `<!-- _class: section -->` + `<!-- _paginate: false -->`
- Navy→Teal グラデーション背景、白文字
- h1 にセクション名、サブタイトルを添える

**注意事項:**
- セクション扉の乱用は避ける（3-5枚に1回が目安）
- 内容を端的に予告する

**前後の組み合わせ:**
- 前: Pattern 3 (section-end) で前セクションを締める
- 後: 本文スライド（Pattern 6-39）

---

### Pattern 3: セクション終了・まとめ

**役割:** セクションの要点を振り返る

**活用シーン:**
- セクションの最後に3-5個のポイントで締めくくる
- 次のセクションへの橋渡し

**デザインポイント:**
- `<!-- _class: section-end -->`
- gray-50 背景、Navy 文字
- h1 + 箇条書きリスト

**注意事項:**
- 箇条書きは3-5項目に抑える
- 新情報は入れず、前のスライドの要約に徹する

**前後の組み合わせ:**
- 前: セクション内の本文スライド
- 後: Pattern 2 (section) で次章開始

---

### Pattern 4: 目次スライド

**役割:** デッキ全体の構成を一覧表示

**活用シーン:**
- タイトルスライドの直後
- 長めのプレゼン（10枚以上）で構成を示す

**デザインポイント:**
- `<!-- _class: toc -->`
- 番号付きリスト、Teal のカウンター装飾
- カスタムボーダー

**注意事項:**
- 項目名は短い名詞句にする（15字以内）
- 7項目以内に収める

**前後の組み合わせ:**
- 前: Pattern 1 (title)
- 後: Pattern 2 (section)

---

### Pattern 5: クロージング

**役割:** プレゼンの最後を飾る

**活用シーン:**
- デッキの最終スライド（必須）
- 感謝の言葉、連絡先の提示

**デザインポイント:**
- `<!-- _class: closing -->` + `<!-- _paginate: false -->`
- Navy→Teal グラデーション背景、白文字
- h1 に "Thank You" 等、h2 に連絡先

**注意事項:**
- シンプルに保つ（テキスト3行以内）

**前後の組み合わせ:**
- 前: Pattern 36 (まとめ) や Pattern 29 (Q&A)

---

## B. カラムレイアウト系

### Pattern 6: 2カラム比較（Before/After）

**役割:** 2つの概念・状態を対比して違いを明確にする

**活用シーン:**
- 導入前 vs 導入後の変化
- 期待 vs 現実のギャップ
- 旧手法 vs 新手法の比較

**デザインポイント:**
- `grid grid-cols-2 gap-6` で均等2分割
- 左パネル: `border-gray-400`（軽いトーン）、右パネル: `border-teal`（強調）
- `bg-gray-50` + `rounded-xl` + `shadow-lg` で浮遊感
- `border-l-4` で左端にアクセントライン

**注意事項:**
- 各カラムの箇条書きは3-5項目に抑える
- 対比する項目数は左右で揃える
- 推奨する側に Teal ボーダーを使う

**前後の組み合わせ:**
- 前: Pattern 2 (section) で問題提起
- 後: Pattern 21 (gradient panel) で結論

---

### Pattern 7a: 2カラム（テキスト＋画像）

**役割:** テキストによる説明と画像を並べて提示する

**活用シーン:**
- プロジェクト概要の説明
- スクリーンショットと説明文の併記
- アーキテクチャ図の解説

**デザインポイント:**
- `grid grid-cols-2 gap-8` で左テキスト、右画像
- 画像エリア: `bg-gray-100` + `rounded-xl` で枠を作る
- テキスト側に h3 サブ見出し + 箇条書き

**注意事項:**
- テキストと画像のバランスを保つ（50:50 が基本）
- 画像が主役なら Pattern 7b（画像左）を使う

**前後の組み合わせ:**
- 前後に自由に配置可能

---

### Pattern 7b: 2カラム（画像＋テキスト）

**役割:** 画像を先に見せてからテキストで説明する

**活用シーン:**
- ビジュアルインパクトを優先するとき
- データの可視化結果を先に見せる

**デザインポイント:**
- Pattern 7a の左右反転
- 画像エリア: `bg-gray-100` + `rounded-xl`

**注意事項:**
- Pattern 7a と同じ

---

### Pattern 8: 3カラム（画像＋テキスト）

**役割:** 3つの要素を並列で紹介する

**活用シーン:**
- 3フェーズのプロセス紹介
- 3つの製品・サービスの概要
- 3部門の説明

**デザインポイント:**
- `grid grid-cols-3 gap-6`
- 各カラム: `bg-gray-50` + `rounded-xl` + `shadow-md`
- 上部に画像エリア + 下部にテキスト

**注意事項:**
- テキストは各カラム3-4行に抑える
- 3つの要素は同格であること

**前後の組み合わせ:**
- 前: Pattern 2 (section) で「3つのXXX」を予告

---

### Pattern 9: 3カラム（アクセントカラー付き）

**役割:** 3つのカテゴリをボーダーカラーで区分して紹介

**活用シーン:**
- カテゴリ別の要素紹介（セキュリティ/パフォーマンス/運用性）
- 優先度や種類の異なる3つの要素

**デザインポイント:**
- `grid grid-cols-3 gap-6`
- `border-t-4` で上部にカテゴリカラー
- Teal / Navy / Gray で3段階の強調度を表現
- h3 に各カテゴリのアクセント色を付ける

**注意事項:**
- 色の使い分けに意味を持たせる（Teal=推奨、Navy=重要、Gray=補助）
- 箇条書きは各3-4項目

---

### Pattern 10: 4カラムレイアウト

**役割:** 4つのフェーズや要素を並列に提示

**活用シーン:**
- プロジェクトの4フェーズ（企画→設計→開発→運用）
- 4つの評価軸
- 四半期ごとの計画

**デザインポイント:**
- `grid grid-cols-4 gap-4`
- `text-sm` で小さめのフォント
- 各カラム: `bg-gray-50` + `rounded-lg` + `shadow-md`

**注意事項:**
- カラム幅が狭いのでテキストは簡潔に
- 各カラム2-3行が上限

---

### Pattern 11: 5カラム（成熟度レベル）

**役割:** 段階的な進化・成長をグラデーションで表現

**活用シーン:**
- 技術成熟度モデル（Level 1-5）
- 組織の成長段階
- 導入フェーズの進行度

**デザインポイント:**
- `.maturity-bar` + `.maturity-level` コンポーネント
- 背景色のグラデーション: gray-100 → #D4EDEE → #8DCDD3 → Teal → Navy
- Level 1-4 は黒文字、Level 5 は白文字
- `.sub` クラスで副題を小さく表示

**注意事項:**
- 各レベルのテキストは1-2行
- レベル名は短い単語にする

---

### Pattern 12: 2x2グリッド（画像＋テキスト）

**役割:** 4つの機能や要素をコンパクトに紹介

**活用シーン:**
- 4つの主要機能の紹介
- 4つの製品カテゴリ

**デザインポイント:**
- `grid grid-cols-2 gap-5`
- 各セル: `bg-gray-50` + `rounded-xl` + `shadow-md` + `flex gap-4`
- 左に小さな画像（100px四方）、右にテキスト

**注意事項:**
- テキストは各セル2-3行
- 画像は統一感のあるサイズで

---

### Pattern 13: 2x3グリッドレイアウト

**役割:** 6つの要素を整然と並べる

**活用シーン:**
- 開発プロセスの6ステップ
- 6つのサービスカテゴリ
- 上段/下段で異なるフェーズを表現

**デザインポイント:**
- `grid grid-cols-3 gap-4`
- `border-l-4` で左ボーダー
- 上段: `border-teal`、下段: `border-navy` でフェーズを区分

**注意事項:**
- 各セルは2-3行のテキスト
- 6項目が上限

---

## C. 縦並びリスト系

### Pattern 14: 縦3つステップ

**役割:** 順序立てたプロセスを上から下に示す

**活用シーン:**
- 3ステップのプロセス
- 実行手順の説明
- 意思決定フロー

**デザインポイント:**
- `.v-step` + `data-step="N"` で番号付き丸を表示
- 左に番号サークル、右にテキスト
- 左ボーダーでステップ間をつなぐ

**注意事項:**
- 3ステップが理想（4以上はスライド分割を検討）
- 各ステップ2-3行

---

### Pattern 15: 番号付きステップ（横型）

**役割:** 左から右への流れでプロセスを示す

**活用シーン:**
- ワークフローの概要
- 3段階のプロセス（分析→設計→実装）

**デザインポイント:**
- `grid grid-cols-5 gap-2` で3ステップ+2矢印
- `.step-number` で番号サークル（Navy/Teal切替可）
- `.step-arrow` で `→` を配置

**注意事項:**
- 3ステップが上限（横幅の制約）
- 各ステップ1-2行のテキスト

---

### Pattern 16: タイムラインレイアウト

**役割:** 時系列の出来事やマイルストーンを表示

**活用シーン:**
- プロジェクトのロードマップ
- 四半期ごとの計画
- 沿革・歴史

**デザインポイント:**
- `.timeline-item` + `.timeline-date` + `.timeline-dot` + `.timeline-content`
- Teal の丸と縦線でタイムラインを視覚化
- 日付は右寄せ、内容は左寄せ

**注意事項:**
- 3-4項目が適切（5以上はスライド分割）
- 日付は「2024 Q1」のように簡潔に

---

### Pattern 17: アイコン付きリスト

**役割:** 各項目にアイコンを付けて視認性を高める

**活用シーン:**
- 製品の特徴一覧
- 技術スタックの紹介
- サービスのメリット

**デザインポイント:**
- `.icon-item` + `.icon-circle`
- アイコンは Unicode 文字（✓ ⚡ 🔒 📊）
- 48px の丸い背景にアイコンを配置

**注意事項:**
- 3-4項目が適切
- 各項目1-2行、太字タイトル + 説明文

---

## D. パネルデザイン系

### Pattern 18: 基本パネル（画像ヘッダー付き）

**役割:** 画像と説明文を組み合わせたカードで情報を提示

**活用シーン:**
- 製品やサービスの紹介
- ブログ記事風のコンテンツ
- ポートフォリオ

**デザインポイント:**
- `bg-white` + `rounded-xl` + `shadow-lg` + `overflow-hidden`
- 上部に画像エリア（160px）、下部にテキスト
- 2カラムのグリッドで横並び

**注意事項:**
- 2枚のパネルが基本（3枚は Pattern 22 を使う）

---

### Pattern 19: 強調パネル（左ボーダー付き）

**役割:** 重要度やカテゴリを左ボーダーの色で示す

**活用シーン:**
- 重要な発見・推奨事項の提示
- 注意事項や補足情報
- カテゴリ別のハイライト

**デザインポイント:**
- `.panel-accent` で Teal 左ボーダー
- Navy ボーダー: `style="border-left:4px solid #1B4565;"`
- Gray ボーダー: `style="border-left:4px solid #9CA3AF;"`
- 縦並びで3段階の情報を提示

**注意事項:**
- Teal=最重要、Navy=推奨、Gray=補足の階層を守る
- 各パネル2-4行

---

### Pattern 20: ガラス風パネル

**役割:** モダンで洗練された印象を与える

**活用シーン:**
- 高級感やテック感を演出したいとき
- グラデーション背景の上に重ねる

**デザインポイント:**
- `.panel-glass`（半透明白、境界線、シャドウ）
- `<!-- _backgroundColor: #E8EDF2 -->` で背景色を設定すると効果的
- 2カラムグリッドで配置

**注意事項:**
- 背景色がないとガラス効果が弱い

---

### Pattern 21: グラデーションパネル

**役割:** 最も重要なメッセージを目立たせる

**活用シーン:**
- 結論のハイライト
- CTA（行動喚起）
- 主要 KPI の強調

**デザインポイント:**
- `.panel-gradient`（Navy→Teal グラデーション、白文字）
- 通常の `.panel` と組み合わせて優先度を表現

**注意事項:**
- 1スライドに1つまで
- 他のパネルと組み合わせて対比を作る

---

### Pattern 22: カード型レイアウト（画像付き）

**役割:** 3つの製品やサービスを均等に紹介

**活用シーン:**
- 製品ラインナップ
- サービスメニュー
- チームメンバー紹介

**デザインポイント:**
- `grid grid-cols-3 gap-5`
- `bg-white` + `rounded-xl` + `shadow-lg` + `overflow-hidden`
- 上部に画像エリア（130px）、下部にテキスト

**注意事項:**
- 各カード2-3行のテキスト
- 画像は統一されたスタイルで

---

## E. 背景・画像系

### Pattern 23: 背景画像全画面

**役割:** インパクトのあるビジュアルで注意を引く

**活用シーン:**
- セクション間の視覚的ブレイク
- コンセプトの象徴的な表現
- キーノートスタイルのスライド

**デザインポイント:**
- `<!-- _class: bg-full -->`
- `<!-- _backgroundImage: "linear-gradient(135deg, #1B4565 0%, #2A5F8F 50%, #3E9BA4 100%)" -->`
- h1 + サブテキスト（テキストシャドウ付き）

**注意事項:**
- テキストは2行以内
- 背景が暗い場合は白文字

---

### Pattern 24: 背景画像右側配置

**役割:** 画像を右に、テキストを左に配置

**活用シーン:**
- プロダクト概要の説明
- ビジュアル+テキストの組み合わせ

**デザインポイント:**
- `![bg right:40%](images/xxx.svg)` Marp の bg ディレクティブ
- 左側にテキスト（h1 + h3 + 箇条書き）

**注意事項:**
- 画像サイズは 40% が標準（30-50% で調整可）
- テキストは左側60%に収める

---

### Pattern 25: 引用スライド

**役割:** 印象的な引用文を大きく表示

**活用シーン:**
- 著名人の言葉を引用
- 顧客の声（テスティモニアル）
- プレゼンの印象的な転換点

**デザインポイント:**
- `<!-- _class: quote -->`
- blockquote で引用文（44px）
- `.attribution` で引用元表示
- 大きな引用符マーク（240px、Teal）

**注意事項:**
- 引用文は1-3文
- 引用元は必ず記載

---

### Pattern 26: 複数画像・分割背景

**役割:** 複数の画像を横並びに配置

**活用シーン:**
- 事業領域の視覚的な表現
- Before/After の画像比較

**デザインポイント:**
- flex レイアウトで均等分割
- `border-radius: 16px` で角丸
- 下部にキャプション

**注意事項:**
- 2-3分割が適切

---

## F. 強調・特殊系

### Pattern 27: 統計強調スライド

**役割:** 数値データを大きく印象的に見せる

**活用シーン:**
- KPI のハイライト
- 調査結果の要約
- 実績の強調

**デザインポイント:**
- `grid grid-cols-3 gap-6`
- `.stat-box` + `.stat-value` + `.stat-label`
- 中央のボックスに `.accent`（グラデーション背景）で最重要指標を強調

**注意事項:**
- 3つの数値が最適（2-4つ）
- 数値は単位付きで簡潔に

**前後の組み合わせ:**
- 前: テキストで文脈を説明
- 後: 数値の意味を解説するスライド

---

### Pattern 28: 中央配置メッセージ

**役割:** 1つのメッセージを大きく中央に表示

**活用シーン:**
- プレゼンのクライマックス
- コンセプトの宣言
- 結論の強調

**デザインポイント:**
- `<!-- _class: center-message -->`
- h1 が 64px で中央表示
- サブテキスト付き

**注意事項:**
- テキストは1-2行
- 連続使用は避ける

---

### Pattern 29: Q&A スライド

**役割:** 質疑応答の時間を示す

**活用シーン:**
- プレゼンの質疑応答セクション

**デザインポイント:**
- `<!-- _class: qanda -->` + `<!-- _paginate: false -->`
- 88px の大きな "Q&A" 見出し
- gray-50 背景

**前後の組み合わせ:**
- 前: 本文の最後のスライド
- 後: Pattern 5 (closing)

## G. 応用パターン系

### Pattern 30: QRコード付き紹介

**役割:** サービスや URL を QR コード付きで紹介

**活用シーン:**
- サービスのダウンロード案内
- URL の共有
- アプリの紹介

**デザインポイント:**
- `grid` で左テキスト + 右 QR エリア（220px固定幅）
